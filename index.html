01

Preparación desde Windows

💾 Respalda TODO lo importante

Antes de tocar particiones, respalda archivos críticos a un disco externo o la nube. Proyectos, configuraciones de VS Code/Cursor, SSH keys, y archivos personales.

⚠ No te saltes esto Un error en el particionado puede borrar datos. Respaldar es no negociable.

📏 Libera espacio en tu disco (SSD/NVMe)

Necesitas mínimo 120GB libres para Ubuntu (recomendado 200GB para tener holgura con contenedores y repos). Desde Windows:

# Abre Disk Management (Win + R → diskmgmt.msc)
# Click derecho en tu partición C: → "Shrink Volume"
# Ingresa el espacio a reducir en MB:
#   200GB = 204800 MB
# Deja el espacio como "Unallocated" — NO crear partición nueva
Nota importante Si Windows no te deja shrinkear suficiente, desactiva temporalmente la hibernación con powercfg /h off (como admin) y reduce los puntos de restauración de System Restore. NO ejecutes defrag en SSD/NVMe — es para discos mecánicos y solo desgasta tu unidad.

⚡ Desactiva Fast Startup

Esto es crítico. Fast Startup bloquea particiones NTFS y causa corrupción cuando Linux intenta leerlas.

# Panel de Control → Opciones de energía → 
# "Elegir el comportamiento de los botones de inicio/apagado"
# → "Cambiar la configuración actualmente no disponible"
# → Desmarca "Activar inicio rápido"

# O por PowerShell (Admin):
powercfg /h off

🔐 Verifica UEFI y apunta tu BitLocker key

Tu PC con Windows 11 ya está en UEFI, pero confirma:

# PowerShell o CMD:
msinfo32
# Busca "BIOS Mode" → debe decir "UEFI"
# Busca "Secure Boot State" → anota si está On u Off

Si tienes BitLocker activo, ve a account.microsoft.com/devices/recoverykey y guarda la recovery key.

03

Configurar BIOS / UEFI

⚙️ Entra al BIOS

Reinicia y presiona la tecla de tu motherboard (DEL, F2, o F12). Ajusta:

  • Secure Boot → Disabled — Simplifica enormemente la instalación de drivers NVIDIA propietarios. Ubuntu soporta Secure Boot con MOK, pero para un primer setup es más limpio desactivarlo.
  • Boot Mode → UEFI (no Legacy/CSM).
  • Boot Order — Pon tu USB primero temporalmente, o usa el Boot Menu (F12) al reiniciar.
  • Fast Boot en BIOS → Disabled — A nivel de firmware, diferente al Fast Startup de Windows.
04

Esquema de Particiones

💿 Layout recomendado para tu disco

Asumiendo ~200GB de espacio libre para Ubuntu, esta es la distribución óptima:

Disco completo (ejemplo 1TB NVMe)
EFI
Windows (NTFS)
Ubuntu / (ext4)
SWAP
/home
EFI — ya existe (~500MB) Windows — tu partición actual / (root) — 120GB ext4 swap — 8GB /home — resto (~72GB) ext4

✦ Opción A: Automático (Recomendado)

Deja que el instalador de Ubuntu use el espacio libre automáticamente. Selecciona "Install alongside Windows Boot Manager" y Ubuntu manejará las particiones. Es la opción más segura para principiantes.

✦ Opción B: Manual

Si quieres control total, crea manualmente: / 120GB ext4, swap 8GB, /home resto ext4. NO crees EFI nueva — usa la existente de Windows con mount point /boot/efi y NO la formatees.

¿Por qué ext4? Ubuntu usa ext4 por defecto: probado en millones de sistemas, rendimiento excelente, y herramientas de recuperación maduras. Si prefieres btrfs (como en Fedora), podés seleccionarlo manualmente en el instalador, pero ext4 es más estable para un primer setup en Ubuntu.
¿Por qué /home separado? Si reinstalas Ubuntu o cambias de distro, tu /home (proyectos, dotfiles, configs) permanece intacto. Solo formateas / y listo.
05

Instalar Ubuntu 26.04 LTS

🚀 Bootea desde la USB

Reinicia con la USB conectada. En el menú de GRUB selecciona "Try or Install Ubuntu". Se cargará el entorno live de Ubuntu.

Nota sobre la gráfica En el live USB probablemente arranque con los drivers open-source nouveau. La resolución puede verse rara o tener glitches — es normal. Los drivers NVIDIA propietarios los instalamos después.

📋 Ejecuta el instalador (Ubiquity/Calamares)

Haz click en "Install Ubuntu" en el escritorio. Sigue estos pasos:

  • Idioma — Español o tu preferencia.
  • Teclado — Latinoamericano o US según tu teclado.
  • Conexión WiFi — Conectate para descargar actualizaciones durante la instalación.
  • Tipo de instalación — Selecciona "Install alongside Windows Boot Manager" para dual boot automático, o "Something else" para particionado manual.

🔨 Configuración de particiones manual

Si elegiste "Something else", en la pantalla de particionado:

  • Verás la partición EFI existente (~500MB, FAT32) y la partición NTFS de Windows. NO toques ninguna de estas.
  • Selecciona el espacio libre ("free space") que creaste en el paso 1.
  • Crea las particiones:

    / — 120GB, filesystem ext4, mount point /
    swap — 8GB (8192MB)
    /home — resto del espacio libre, filesystem ext4, mount point /home
  • Para el EFI: selecciona la partición EFI existente y asígnale /boot/efi. NO la formatees — solo asigna el mount point.
⚠ Verificación final antes de instalar Revisa que las particiones de Windows digan "preserve" o "no change" y que SOLO se creen/formateen las particiones nuevas en el espacio libre. Si ves "format" en una partición de Windows, PARA y revisa.

👤 Crea tu usuario

Configura tu nombre de usuario y contraseña. Marca la opción de hacerte administrador (acceso a sudo). Ubuntu no usa root directo — todo va por sudo.

Click en "Continue" y espera ~15-20 minutos. Cuando termine, reinicia y remueve la USB.

Primer arranque y configuración base

🖥️ Primer login en Ubuntu 26.04 LTS

GRUB debería aparecer mostrando Ubuntu y Windows. Selecciona Ubuntu. Abre una terminal (Ctrl + Alt + T) y actualiza:

# Actualiza la lista de paquetes e instala actualizaciones
sudo apt update && sudo apt upgrade -y

# Instala herramientas esenciales que no vienen por defecto
sudo apt install -y build-essential curl wget git vim htop

# Reinicia después de actualizar (puede incluir kernel nuevo)
sudo reboot

📦 Habilita repositorios adicionales

Ubuntu tiene repositorios "restricted", "universe" y "multiverse" que contienen software adicional:

# Habilita universe, multiverse y restricted (si no están activos)
sudo add-apt-repository -y universe
sudo add-apt-repository -y multiverse
sudo add-apt-repository -y restricted

# Actualiza repos
sudo apt update

🎬 Codecs multimedia

Para videos, música y contenido web sin fricción:

# Codecs multimedia completos (incluye patentes restrictivas)
sudo apt install -y ubuntu-restricted-extras

# Soporte VA-API para NVIDIA (aceleración de video por hardware)

Drivers NVIDIA + CUDA (rendimiento máximo)

🟢 Instala los drivers propietarios de NVIDIA

Este es el paso más importante para tu RTX 4060. Ubuntu tiene una herramienta oficial:

# Ver qué drivers recomienda Ubuntu para tu GPU
ubuntu-drivers devices

# Instala el driver recomendado automáticamente
sudo ubuntu-drivers autoinstall

# O instala manualmente el último driver disponible:
# sudo apt install -y nvidia-driver-580

# Reinicia
sudo reboot
¿Cuál driver elegir? ubuntu-drivers autoinstall instala el driver recomendado y probado para tu hardware. Si quieres el último (a veces con mejor soporte para CUDA reciente), busca la versión más alta con ubuntu-drivers devices e instálala manualmente.

✅ Verifica la instalación

Después de reiniciar:

# Verifica que el driver cargó
nvidia-smi
# Deberías ver tu RTX 4060, driver version, CUDA version

# Verifica Vulkan (crítico para rendimiento gráfico)
sudo apt install -y vulkan-tools
vulkaninfo | head -20

# Verifica que Nouveau está bloqueado (no debe mostrar nada)
lsmod | grep nouveau

🧪 CUDA Toolkit para desarrollo

Para compilar con CUDA necesitas el toolkit completo:

# Instala CUDA toolkit desde los repos de Ubuntu
sudo apt install -y nvidia-cuda-toolkit

# Verifica instalación
nvcc --version

# Agrega al PATH si es necesario:
echo 'export PATH=/usr/local/cuda/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH' >> ~/.bashrc
source ~/.bashrc
Para tu stack de IA ligero Ollama y Whisper usan CUDA runtime, no necesitan nvcc. Con los drivers NVIDIA propietarios ya tienes lo necesario para la mayoría de workloads. El toolkit completo solo lo necesitás si compilás proyectos nativos con CUDA.

🚀 Optimización NVIDIA para Wayland

Ubuntu 26.04 usa Wayland por defecto en GNOME. Activa KMS y parámetros de rendimiento:

# Activa kernel mode setting (obligatorio para Wayland + NVIDIA)
sudo sed -i 's/GRUB_CMDLINE_LINUX_DEFAULT="[^"]*"/GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nvidia-drm.modeset=1"/g' /etc/default/grub

# Actualiza GRUB
sudo update-grub

# Reinicia
sudo reboot
08

Optimización del sistema — Velocidad extrema

⚡ zram ya está activo (no toques)

Ubuntu configura zram por defecto desde hace varias versiones: swap comprimido en RAM que es más rápido que swap en disco y no desgasta tu NVMe. No necesitas crear swap file adicional a menos que quieras hibernar.

# Verifica que zram está activo
swapon --show
# Debe mostrar /dev/zram0

# Si quieres ajustar el tamaño (opcional):
# Por defecto usa ~50% de tu RAM (16GB en tu caso).
# Está bien así para 32GB.

🐧 Kernel tweaks para latencia baja

Para un sistema de desarrollo súper responsivo:

# Editor de parámetros del kernel
sudo nano /etc/sysctl.d/99-performance.conf

# Pega esto:
vm.swappiness=10
vm.vfs_cache_pressure=50
vm.dirty_ratio=15
vm.dirty_background_ratio=5
kernel.sched_latency_ns=1000000
kernel.sched_min_granularity_ns=100000
kernel.sched_wakeup_granularity_ns=50000

# Aplica
sudo sysctl --system

# Scheduler I/O para NVMe (mq-deadline o none)
# Verifica cuál usas:
cat /sys/block/nvme0n1/queue/scheduler
# Si dice "none", está bien (NVMe maneja su propia cola eficientemente).

🎨 GNOME — Ajustes de rendimiento

  • Desactiva animaciones: Settings → Accessibility → Seeing → Animation → Off. Las animaciones de GNOME consumen recursos y no aportan nada a la productividad.
  • Extensiones útiles: Instala gnome-tweaks y la extensión Dash to Dock para un dock más eficiente.
  • Tiling: Ubuntu 26.04 incluye soporte de tiling nativo en GNOME. Activálo en Settings → Multitasking → Tiling.
  • Wayland vs X11: GNOME en Wayland funciona excelente con NVIDIA en Ubuntu 26.04. Si tienes glitches raros, cambia a X11 desde la pantalla de login (esquina inferior derecha).

Configurar GRUB y flujo de dual boot

🔄 Verifica que Windows aparezca en GRUB

Ubuntu detecta Windows automáticamente vía os-prober. Si no aparece:

# Instala os-prober
sudo apt install -y os-prober

# Edita configuración de GRUB
sudo nano /etc/default/grub
# Asegurate de que NO tenga: GRUB_DISABLE_OS_PROBER=true
# O si lo tiene, cámbialo a: GRUB_DISABLE_OS_PROBER=false

# Regenera GRUB en la ruta correcta de Ubuntu
sudo update-grub

# Reinicia y verifica
sudo reboot
⚠ NUNCA edites /boot/grub/grub.cfg manualmente Ese archivo se regenera automáticamente. Si lo editás a mano, perderás los cambios en la próxima actualización de kernel. Siempre editá /etc/default/grub y ejecutá sudo update-grub.

⏱️ Ajusta timeout y OS default

sudo nano /etc/default/grub

# Tiempo de espera
GRUB_TIMEOUT=5

# Ubuntu como default (ya lo es generalmente)
GRUB_DEFAULT=0

# O que recuerde tu última elección:
# GRUB_DEFAULT=saved
# GRUB_SAVEDEFAULT=true

# Aplica
sudo update-grub

🕐 Arregla el reloj entre Windows y Linux

Windows usa localtime, Linux usa UTC. Solución simple:

# En Ubuntu, usa localtime como Windows:
sudo timedatectl set-local-rtc 1

# Alternativa (hacer que Windows use UTC) — ejecutar en CMD Admin de Windows:
# reg add "HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\TimeZoneInformation" /v RealTimeIsUniversal /d 1 /t REG_DWORD /f

🎮 Flujo rápido: reiniciar a Windows desde Ubuntu

# Reinicia directo a Windows (una sola vez)
sudo grub-reboot "Windows Boot Manager" && sudo reboot

# O usa el índice numérico (más robusto):
# Verifica el índice con:
grep -n "Windows" /boot/grub/grub.cfg
# Si aparece en la línea 150 (menuentry 4), usá:
# sudo grub-reboot 4 && sudo reboot

🔧 Troubleshooting común

Pantalla negra después de instalar NVIDIA Ve a TTY con Ctrl + Alt + F2, login, y ejecuta sudo ubuntu-drivers autoinstall && sudo reboot. Si persiste: sudo dmesg | grep -i nvidia.
Windows no aparece en GRUB Verifica que GRUB_DISABLE_OS_PROBER=false en /etc/default/grub y regenera con sudo update-grub.
WiFi no funciona Identifica tu chip con lspci | grep -i network. Intel funciona out-of-the-box. Broadcom puede necesitar broadcom-sta-dkms. Realtek varía según modelo.
Docker no ve la GPU sudo nvidia-ctk runtime configure --runtime=docker && sudo systemctl restart docker. Test: docker run --rm --gpus all nvidia/cuda:12.6.0-base-ubuntu22.04 nvidia-smi.
Wayland con NVIDIA se siente lento Asegúrate de que nvidia-drm.modeset=1 esté en los parámetros del kernel (cat /proc/cmdline). Si persiste, prueba la sesión X11 desde la pantalla de login.

🎯 Checklist final de rendimiento

  • zram activo: swapon --show → debe mostrar /dev/zram0.
  • NVIDIA KMS: cat /proc/cmdline | grep nvidia-drm → debe incluir nvidia-drm.modeset=1.
  • Driver NVIDIA cargado: nvidia-smi → muestra RTX 4060.
  • Vulkan funcional: vulkaninfo --summary → muestra tu GPU.
  • Docker + GPU: contenedor de test muestra nvidia-smi.
  • Timeshift configurado: snapshot inicial creado.
